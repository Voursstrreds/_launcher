LAUNCHER — PROGRAM FLOW DIAGRAM
================================

Each numbered box corresponds to a test directory under Tests/.

The Python pipeline (steps 3a–3f) is rule-set-agnostic. Every stage
dispatches through Rules.ACTIVE_RULE_SET — a RuleSetDescriptor that
bundles the syntactic side (schemas) with the semantic side (builders)
for one launcher rule-set. Two rule-sets currently exist:

    GENERIC_TASK_LAUNCHER   Entry / Group        -> .service / .target
    PODMAN_COMPOSE          Service / Network    -> .service / .service

ACTIVE_RULE_SET is selected at the top of Rules.py.

         |        register signal listeners (Reloading, UnitNew, UnitRemoved).

         |   | 3a. INPUT VALIDATION          |   Validator.py: load_and_validate()
         |   |                               |   Schemas.py: per-rule-set Cerberus dicts
         |   |                               |   Rules.py:   ACTIVE_RULE_SET.schemas dispatch
         |   |                               |   - load_raw: YAML safe_load

         |   |                               |     int->str, Path->abspath
         |   |                               |   - validate_types: Cerberus check against
         |   |                               |     ACTIVE_RULE_SET.schemas[discriminator]

         |   |                               |     over ref_fields
         |   +-------------------------------+   Test dir: Tests/InputValidation/

         |                  v
         |   +-------------------------------+
         |   | 3b. RELATIONSHIP MAPPING      |   RelationshipMapping.py:
         |   |                               |   - compute_before: invert dependency graph
         |   |                               |   - compute_group_maps: reconcile group/member
         |   |                               |     (or network/network_members for podman)
         |   +-------------------------------+   Test dir: Tests/RelationshipMapping/
         |                  |
         |                  v
         |   +-------------------------------+
         |   | 3c. DAG VALIDATION            |   RelationshipMapping.py:
         |   |                               |   - validate_dag: DFS cycle detection on
         |   |                               |     dependency graph and group membership
         |   |                               |     graph separately
         |   |                               |   - compute_unified_before: merge before_map
         |   |                               |     + group_of into single structure
         |   |                               |   - validate_unified_dag: DFS cycle detection
         |   |                               |     on merged before-after graph
         |   |                               |   GraphVisualizer.py: emits DOT + PNG renders

         |   |                               |     unified_graph into Results/

         |                  v
         |   +-------------------------------+
         |   | 3d. COMMAND GENERATION        |   CommandGenerator.py: build_all()
         |   |                               |   Builders.py: ACTIVE_RULE_SET.command_builder
         |   |                               |   - build_generic_task_launcher_command, or
         |   |                               |   - build_podman_compose_command
         |   |                               |   Each orchestrator computes maps, validates
         |   |                               |   the DAG, draws graphs, then delegates to
         |   |                               |   construct_*_command for pure construction.
         |   |                               |   Output is a list of GeneratedCommand /
         |   |                               |   PodmanGeneratedCommand instances carrying
         |   |                               |   all fields needed downstream.
         |   +-------------------------------+   Test dir: Tests/CommandGeneration/
         |                  |
         |                  v
         |   +-------------------------------+
         |   | 3e. UNIT FILE GENERATION      |   UnitFileCreator.py: generate_all()
         |   |                               |   Builders.py: ACTIVE_RULE_SET.unit_file_builder
         |   |                               |   - create_generic_task_launcher_unit_file:
         |   |                               |     Entry -> [Unit]+[Service]+[Install],
         |   |                               |     Group -> [Unit]+[Install] (no [Service]),
         |   |                               |     writes .service / .target files.
         |   |                               |   - create_podman_compose_unit_file:
         |   |                               |     Service -> [Unit]+[Service]+[Install]
         |   |                               |       with podman run as ExecStart and
         |   |                               |       podman stop / podman rm teardown,
         |   |                               |     Network -> Type=oneshot RemainAfterExit
         |   |                               |       with podman network create / rm,
         |   |                               |     writes .service files.
         |   |                               |   Both: merge before+group -> Before/RequiredBy,
         |   |                               |   merge after+members -> After/Requires,
         |   |                               |   build key_map from all instances.
         |   +-------------------------------+   Test dir: Tests/UnitFileGeneration/
         |                  |
         |                  v
         |   +-------------------------------+
         |   | 3f. MANIFEST GENERATION       |   ManifestWriter.py: write_manifest()
         |   |                               |   Builders.py: ACTIVE_RULE_SET.manifest_builder
         |   |                               |   - build_generic_task_launcher_manifest_entry:
         |   |                               |     Entry -> ENTRY, Group -> GROUP
         |   |                               |   - build_podman_compose_manifest_entry:
         |   |                               |     Service -> SERVICE, Network -> NETWORK
         |   |                               |   Writes INSTANCE...END blocks with all
         |   |                               |   fields: key, name, unit_file_name, type,
         |   |                               |   path, command, after, before, group,
         |   |                               |   members, order, failure_behavior,
         |   |                               |   dependency_behavior. List fields are
         |   |                               |   space-separated; empty lists produce
         |   |                               |   empty values.
         |   +-------------------------------+   Test dir: Tests/ManifestGeneration/
         |
         |   Integration test (3a->3f):         Test dir: Tests/ParserIntegration/
         |   Full pipeline from raw YAML through all stages.
         |   Valid cases: 20 (minimal -> 19-node stress).
         |   Invalid cases: 15 (validation, reference, dep/group/unified cycles).
         |
         |   UnitGenerator exits. Parent continues.
